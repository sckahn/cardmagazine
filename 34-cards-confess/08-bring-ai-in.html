CHAPTER 07  ·  SaaS LLM과 사내 LLM 사이
TO‑BE BRING AI INSIDE데이터를 보내지 말고 도구를 받아오기 · 사내 LLM 로드맵

데이터를 보내지 말고 —
도구를 받아오십시오.

SaaS LLM · 외부 서비스
ChatGPT · Claude · Gemini
  • 가입만 하면 즉시 강력. 도입 비용이 낮습니다.
  • 데이터는 ToS · 옵트아웃 · 벤더 정책에 의존합니다.
  • 누출이 발생해도 경계는 외부에 있습니다.
사내 LLM · 내부 인프라
사설망(VPC) · 자체 서버(On-prem) · 단말 내장(On-device)
  • 초기 인프라·운영 비용이 듭니다. 시간이 필요합니다.
  • 데이터·로그·접근권 모두 자사 통제입니다.
  • 책임도 자사 — 그래서 경계가 우리 손 안에 있습니다.
PRINCIPLE · 데이터 OUT → AI 도구 IN
데이터를 보내는 대신, 업무 방식·도메인 지식·사람의 아이디어로 추상화한 요청만 외부에 보냅니다. AI는 도구·코드·설계를 만들어 돌려보내고, 사내에서 검토 후 실행합니다.
사내 게이트웨이
승인된 LLM만 통과 · 자동 마스킹·감사.
아이디어 → 도구
패턴만 설명 → 스크립트·쿼리 받아 사내 실행.
사설망(VPC)
Azure OpenAI · Bedrock · Vertex 사설망.
사내 외부 자료 검색(RAG)
자체 벡터 DB · 검색 결과만 답변에 인용.
TO‑BE · 사내 LLM 구축 로드맵
PHASE 1 · 기반
사설 추론 인프라
사내 GPU 클러스터 또는 VPC 전용 엔드포인트.
PHASE 2 · 모델
자체 호스팅 LLM
Llama·Qwen·Solar 같은 오픈 가중치 사내 운영.
PHASE 3 · 도메인
사내 코파일럿
자사 데이터로 적합화한 작은 모델·RAG가 직원 곁에 머무릅니다.
UNTIL THEN · 그때까지 사내 LLM이 자리 잡을 때까지 — 새 도구를 향한 신나는 마음은 잠시 가라앉혀 주십시오. 외부 LLM에 회사 데이터를 보내는 대신, 우리는 도구와 기능을 만들어 안으로 들이는 데 집중합니다.
